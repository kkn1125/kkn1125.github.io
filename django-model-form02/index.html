기존 정보에 수정된 정보를 덮어쓰기

회원정보를 수정하다보면 초기에는 request.POST['datatype']으로 하나씩 받아와 데이터베이스를 조회해서 user를 가져오고 하나 씩 담았습니다.

1def update(request, num):
2    user = User.objects.filter(pk=num).get(pk=num)
3
4    nickname = request.POST['nickname']
5    email = request.POST['email']
6    password = request.POST['password']
7    profile = request.POST['profile']
8
9    user.nickname = nickname
10    user.email = email
11    user.password = password
12    user.profile = profile
13
14    user.save()

위의 형식이 꽤 번거로웠습니다. 스프링에서는 form data를 전송하면 request를 보낼 때 알아서 객체에 담아줬습니다. 분명 장고에도 있을 거라는 예감은 틀리지 않았습니다.

모델 폼

modelForm을 쓰다보면 create, update가 단 몇 줄에 끝납니다. 그렇게 편하게 쓰던 중에 고민이 하나 생겼습니다.

공백은 무시하고 작성한 데이터만 받기

게시글을 수정할 때는 모든 데이터를 받아 업데이트 폼으로 연결하면 해당 데이터들이 pk값 빼고는 다 나타나는 구조였습니다.

1from django.db import models
2from django.utils import timezone
3class Post(models.Model):
4    num = models.AutoField(primary_key=True)
5    title = models.CharField(max_length=45, null=False, unique=True)
6    content = models.CharField(max_length=1000, null=False)
7    author = models.CharField(max_length=45, null=False)
8    regdate = models.DateTimeField('created', default=timezone.now, editable=False, null=False, blank=False)
9    updates = models.DateTimeField('updated', default=timezone.now, editable=False, null=False, blank=False)

대략 위와 같은 컬럼은 가진 모델로 작성했습니다. 그런데 만일 변경하지 않아도 되는, 즉, 기존 값을 유지하되 새로운 값이 있으면 덮어써야 하는 경우가 발생하는 때가 있습니다.

예를들면 이미지같은 경우입니다.

1class Post(models.Model):
2    # 코드 상동
3    """
4    이미지 필드 추가
5    """
6    cover = models.ImageField(upload_to="", null=True, blank=True)
7    # ...

이렇게 이미지 필드가 추가된 경우에는 html input태그의 file타입 특성상 value값 설정이 안 됩니다.

태그에 value를 지정해도 form데이터에 넘겨도 보안상 이유로 값이 넘어가지 않습니다.

그리고 cover 필드 추가와 함께 저는 forms.py에 작성된 패스워드 필드와 커버 필드에 속성을 변경했습니다.

1class UserForm(ModelForm):
2    """ 추가 """
3    cover = ImageField(widget=FileInput(attrs={'type': 'file'}))
4    password = CharField(widget=PasswordInput(attrs={'type': 'password', 'autocomplete': 'current-password'}))
5    """ 추가 """
6    class Meta:
7        model = User
8        fields = [
9            'profile', 'nickname', 'email', 'password'
10            ]
11        widgets = {
12            'email': EmailInput(attrs={'autocomplete': 'username'}),
13        }

cover type을 또 지정해 준 이유는 django에서 기본적으로 fileInput에 현재와 변경 정보를 함께 띄워줍니다. 그게 싫어서 file input만 나타나게 하기 위함입니다.
password는 charfield이지만 암호화되어 보여야하니 type을 변경한 것 입니다.

1import .models from User
2import .forms from UserForm
3import rest_framework.decorators from api_view
4
5@api_view(['GET', 'POST'])
6def update(request, num):
7    user = User.objects.filter(num=num).get(num=num)
8
9    if request.method == 'GET':
10
11        context = {
12            'path_type': path_type(request),
13            'userForm': UserForm(instance=user),
14        }
15
16        return render(request, 'scheduler/signup.html', context)
17
18    else:
19        userForm = UserForm(request.POST or None,
20                            request.FILES or None,
21                            instance=user)
22
23        if userForm.is_valid():
24            uf = userForm.save(commit=False)
25            request.session['sign'] = uf.pk
26            uf.save()
27
28        return redirect('account:update', user.pk)

위 코드처럼 UserForm에 instance와 request를 넣어줬습니다. cover라는 field도 추가를 했구요. 이대로 form을 작성하고 post요청하게 되면 데이터가 변경/저장 되지 않습니다. is_valid에서 Falsy한 값을 받아 save되지 않게 됩니다.

이유는 request.FILES['profile']이 빈 값이고, 비밀번호는 password타입으로 변경하면 instance에 유저 정보를 넣고 form을 출력해도 value에 회원 비밀번호가 들어가지 않습니다.

이렇게 빈 값이 발생하면 is_valid에서 막히게 되는데 찾아보니 간단하게 해결할 수 있는 문제였습니다.

forms.py에 있던 password와 cover에 requied를 False로 변경해줍니다.

1# forms.py
2class UserForm(ModelForm):
3    cover = ImageField(required=False, widget=FileInput(attrs={'type': 'file'}))
4    password = CharField(required=False, widget=PasswordInput(attrs={'type': 'password', 'autocomplete': 'current-password'}))
5
6    # Meta ...

이제 cover는 필수로 받는 항목이 아니기 때문에 이미지 파일을 새로 업로드 하지 않아도, 빈값으로 form data가 전송되어져도 기존의 값 그대로 유지 됩니다.

하지만 password는 아닙니다. 무엇 때문인지 자세히 모릅니다. 어디까지나 추측입니다만, type에 password가 설정되면서 value에 값이 들어가지 않는 점이 문제라 생각됩니다.

즉, 애초에 password type의 input은 보안상 value에 값이 들어가지 않으니 form data가 POST로 전송된다면, 일반적으로 패스워드는 로그인에 사용하기 때문에 필수 항목으로 보고 가져와 적용하는 것으로 생각됩니다. 자세한 원리는 찾아서 포스팅 내용을 수정하도록 하겠습니다. 어디까지나 제 생각입니다.

그래서 cover는 required를 False로 설정하면 해결됩니다. password는 is_valid하기 전에 User모델에서 pk값으로 가져온 데이터의 비밀번호를 변수에 담아두고 입력된 비밀번호가 빈 값일 때 저장해두었던 변수를 대신 저장시키는 방식으로 하면 됩니다.

1# 생략
2
3@api_view(['GET', 'POST'])
4def update(request):
5    # 생략
6    user = User.objects.filter(num=num).get(num=num)
7    origin_password = user.password         # +
8    
9    if request.method == 'GET':
10        # 생략
11
12    else:
13        # 생략
14
15        if userForm.is_valid():
16            uf = userForm.save(commit=False)
17            if not uf.password:             # +
18                uf.password = origin_pass   # +
19
20            # 생략
21
22            uf.save()
23        return redirect('account:update', user.pk)

찾아보면 분명히 간결하고 편리한 방법이 있을 것이라 생각합니다. 포스팅에 포함된 추측성 내용은 추후에 알게되면 정정해서 다시 게재할 예정입니다.
