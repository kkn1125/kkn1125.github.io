JDBC Template 사용법

오랜만에 JAVA를 사용했습니다. 요즘 만들고 싶은게 많아서 js를 쭉 쓰다보니 점점 JAVA를 잊어가는 느낌입니다.

MyBatis를 spring-boot에서는 간단하게 사용했었지만 spring에서는 Mybatis를 구현해 본 적이 없어서 이것저것 하던 중 jdbc template도 한 번 써봐야겠다 싶어서 사용법만 기록하려합니다.

RootConfig 설정

1@Configuration
2@EnableWebMvc
3@ComponentScan(basePackages = {"com.test.web"})
4public class RootConfig {
5	@Bean
6	public DataSource dataSource()
7	{
8		DriverManagerDataSource mysqlConfig = new DriverManagerDataSource();
9		mysqlConfig.setDriverClassName("com.mysql.jdbc.Driver");
10		mysqlConfig.setUrl("jdbc:mysql://localhost:3306/kimsontest");
11		mysqlConfig.setUsername("kimsontest");
12		mysqlConfig.setPassword("kimsontest");
13		return mysqlConfig;
14	}
15}

RootConfig로 db를 연결하고 datasource를 받아 사용하려합니다.

template 사용전

1@Repository
2public class UserDao implements DAOImpl<User>
3{
4   @Autowired
5	DataSource source;
6	
7	@Override
8	public List<User> getUserList() {
9      Connection con = null;
10      PreparedStatement st = null;
11      ResultSet rs = null;
12      String sql = null;
13      List<User> list = new ArrayList<User>();
14      try {
15         con = source.getConnection();
16         sql = "SELECT * FROM user";
17         st = con.prepareStatement(sql);
18         rs = st.executeQuery();
19         while(rs.next()) {
20            User user = new User();
21            user.setNum(rs.getInt("num"));
22            user.setNamez(rs.getString("namez"));
23            user.setAge(rs.getInt("age"));
24            user.setId(rs.getString("id"));
25            user.setPw(rs.getString("pw"));
26            list.add(user);
27         }
28      } catch(Exception ex) {
29         ex.getStackTrace();
30      } finally {
31         try {
32            if(con != null)con.close();
33            if(st != null)st.close();
34            if(rs != null)rs.close();
35         } catch(Exception ex) {
36            ex.getStackTrace();
37         }
38      }
39      return list;
40   }
41}

template을 쓰기 전에는 각 메서드를 만들때마다 연결하고 연결을 끊고 하는 구문이 필요한 기능만큼 많아져서 때로는 수정 할 때 어디를 빼먹었고 어디서 오류나는지 많이 애를 먹었습니다.

그런데 template을 사용하면서 느낀 것은 코드량이 많이 줄었다는 것을 바로 느낄 수 있었습니다.

template 사용 후

1@Repository
2public class UserDao implements DAOImpl<User>
3{
4   @Autowired
5	DataSource source;
6	
7	@Override
8	public List<User> getUserList() {
9      JdbcTemplate jdbcTemplate = new JdbcTemplate(source);
10      // dataSource를 인자로 넘겨줍니다.
11		List<User> list = jdbcTemplate.query("SELECT * FROM user", new Object[] {}, new RowMapper<User>() {
12			@Override
13			public User mapRow(ResultSet rs, int rowNum) throws SQLException {
14				User u = new User();
15				u.setNum(rs.getInt("num"));
16				u.setNamez(rs.getString("namez"));
17				u.setAge(rs.getInt("age"));
18				u.setId(rs.getString("id"));
19				u.setPw(rs.getString("pw"));
20				return u;
21			}
22		});
23		return list;
24   }
25}

여전히 rs를 통해 가져와서 VO에 담고 데이터를 넘겨주는 것은 번거롭지만(Mybatis에 비해) 훨씬 간결해진 모습입니다.

JPA를 배워서 나중에는 MyBatis 등 여러가지를 비교하는 포스팅을 할 예정입니다. 많이 먼 이야기지만요...

1@ComponentScan(basePackages = {"com.test.web"})
2@Controller
3@RequestMapping("/")
4public class HomeController {
5	
6	@Autowired
